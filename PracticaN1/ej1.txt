1) HTML es un lenguaje de marcado. Sirve para describir semánticamente documentos e incluso aplicaciones.
Fue creado en 1990.
Existen las versiones HTML 3.O, HTML 3.2, HTML 4.01, XHTML 1.O, XHTML 2.0.
Hoy en día, la versión actual es HTML5

2) Las tecnologías deben ser compatibles con versiones anteriores.
Las especificaciones y las implementaciones deben coincidir incluso si esto significa cambiar la especificacion en lugar de las implementaciones.
Las especificaciones deben ser lo suficientemente detalladas como para que las implementaciones interoperabilidad completa sin ingenieria reversa entre si.

3)Los tags y atributos desaprobados son parte del estándar HTML (lenguaje de marcas hipertextual) cuyo uso ya no es recomendado.
Una etiqueta o atributo obsoleto es un elemento que está desaconsejado para nuevos desarrollos porque existe una forma más moderna, semántica y accesible de lograr el mismo resultado

5)El HTML permite a los autores especificar metadatos -- información sobre un documento más que contenido del propio documento -- de diferentes de maneras.
En general, la especificación de metadatos implica dos pasos:
1- Declaración de una propiedad y de un valor para esta propiedad. Esto puede hacerse de dos maneras:
        - Desde dentro de un documento, por medio del elemento META.
        - Desde fuera de un documento, vinculando los metadatos por medio del elemento LINK (véase la sección sobre tipos de vínculos).
2- Referencia a un perfil en el que se definen la propiedad y sus valores legales. Para designar un perfil, se usa el atributo profile del elemento HEAD.
